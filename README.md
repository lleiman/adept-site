# ADEPT — AI Production Studio

Сайт-портфолио ADEPT: AI + CG продакшн для брендов.

## Структура

- `server.js` — Node/Express: статика + edit API + загрузка картинок
- `index.html` — главная: hero, тэглайн, асимметричная карусель из 17 кейсов, CTA
- `case.html` — детальная страница кейса (hash-routing: `#dragon`, `#rosatom`…)
- `portfolio.html` — список всех проектов
- `edit-mode.js` / `edit-mode.css` — общий inline-редактор
- `content.json` — текстовые и картиночные оверрайды (создаётся сервером)
- `uploads/` — загруженные через редактор картинки

## Локально

```bash
npm install
npm start                  # → http://localhost:3000
ADMIN_PASSWORD=ADEPT npm start   # переопределить пароль (по умолчанию "ADEPT")
```

## Деплой

Railway подхватывает `npm start` → `node server.js`. `PORT` пробрасывается из окружения.

**Чтобы правки не пропадали между деплоями** — нужен Railway Volume:

1. В Railway dashboard: проект → сервис `adept-site` → вкладка **Volumes** → **Create Volume**, mount path: `/data`
2. На вкладке **Variables** добавь:
   - `DATA_DIR=/data`
   - (опционально) `ADMIN_PASSWORD=твой_новый_пароль` — иначе по умолчанию `ADEPT`
3. После сохранения Railway автоматически передеплоит. `content.json` и `uploads/`
   будут писаться в `/data`, который выживает после редеплоя.

Без volume сервер пишет в корень контейнера — переживает текущий deploy, но
сбрасывается при следующем (например, после git push).

## Редактирование контента

- **⌘E / Ctrl+E** — переключает режим редактирования
- При первом входе попросит пароль администратора (по умолчанию `ADEPT`,
  кэшируется на сессию в `sessionStorage`)
- Клик по тексту → редактируешь прямо в макете → Enter / клик вне сохраняет
- При наведении на картинку появляются кнопки **Заменить** / **Сбросить**
- Все правки летят в `/api/content` и пишутся в `content.json` в корне проекта
- Загруженные картинки уходят в `/uploads/` и линкуются по URL `/uploads/<file>`
- На главной редактируется: hero, тэглайн, все 17 кейсов (eyebrow + title + img)
- В `/case` редактируется: title, lead, описание (по абзацам), статистика,
  команда (group + строки), постер, галерея
- В `/portfolio` редактируется: заголовок страницы + те же 17 карточек

**Когда настроен Volume** — правки переживают редеплои (см. секцию «Деплой»).
Без volume — только до следующего деплоя.

## Выход / сброс

- `ESC` или `⌘E` — выход из режима
- Кнопка «Сбросить всё» в нижней панели стирает `content.json` целиком
